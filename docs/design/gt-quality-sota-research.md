### (1) Top-5 highest-EV ideas

**1. [Score 7.5] Wilson-CI / boundary-stratified re-grading as the prerequisite gate — implement P1.3 before any repair-math investment.**
- **What:** 8-crop Bernoulli grading has SE≈0.17 against the 0.4 D/E pivot (adversarial plan §1 insight #5). No fancier repair operator (CPD, SDF, MRF, graph-cut) can be validated as "it worked" while the re-grade that judges it is itself a coin flip frozen only by `seed=0`.
- **Integration sketch:** `tools/labelqc/crop_qc.py` — Wilson CI on `pap_med`/`ridge_med`/`frac_crops_good`; tier assigned only when CI excludes the nearest threshold, else `"borderline"` → adaptively double crops (8→16→32, cheap since WAN-local); add a 10–30%-valid boundary stratum (currently unsampled — margins where `auto_grown` damage concentrates are systematically excluded from the alignment axis today). Replace `repair_corpus.py`'s `improved = a_ridge > b_ridge+5` (~1σ) with a CI-separated gate.
- **Prerequisite:** P0.2 (emit `n_offs`, kill fail-open `iqr=0` on low-n crops) must land first — Wilson CIs on an `iqr` that silently reports 0 are meaningless.
- **Effort:** ~1 day. **Impact:** high — this is the gating condition for trusting every other measurement in the system, including the fault-injection experiment below.

**2. [Score 6] Fault-injection round-trip promoted from one-off validation to a standing per-model-generation gate, with an explicit non-circularity tripwire before Loop 2.**
- **What:** The adversarial plan §3 already specifies a rigorous one-time experiment: 5 known-good segments, 4 synthetic corruptions (wrong-wrap shift, uniform offset, smooth warp, local scramble), full round-trip through crop_qc→scorecard→repair_corpus→re-grade, with hard pass criteria (critically, (a) wrong-wrap-shift must NOT grade above the original — under current code it will, at pap≈100/iqr≈0, which is the single measurement proving the whole finding set).
- **What's actually new here (not already in the plan):** turning this into a re-run-on-every-model-generation gate, and — more importantly — using it as the **non-circularity tripwire for Loop 2** (model-in-the-loop label-audit): a new teacher/student generation is only allowed to relabel real GT once it reproduces clean separation on synthetic corruptions with known ground truth. This is a currently-missing piece of the model-in-the-loop safety story that the improve-doc flags as an open risk ("model-in-the-loop flywheel, circularity risk") but never resolves with a concrete gate.
- **Integration sketch:** wrap §3's script as a fixture in `tools/labelqc/`, triggered whenever `label-audit`'s underlying model changes; gate Loop 2's write-back on it passing. Must itself use CI-aware pass/fail (idea #1) once standing, or it becomes a second flaky gate.
- **Effort:** hours to wrap (the experiment itself is already scoped at half a day, zero new code beyond P0). **Impact:** high for the Loop-2 safety property specifically; the "run before shipping" framing for P0/P1 changes is process discipline, not new capability — the real marginal value is the Loop-2 tripwire.

**3. [Plan cross-reference, not independently judged] P0.1–P0.3: single source of truth + kill pap-alone tiering + fail-open iqr fix.**
- These three are structural bugs, not research ideas, but they dominate expected value because everything above depends on them. Currently three divergent grading functions (`grade_corpus.py`, `scorecard.py`, `repair_corpus.py`'s inline `a_iqr<=4→A`) read/write colliding scorecard files and disagree on the same segment depending on run order; `pap` alone (a one-sided, single-voxel, hardcoded-threshold, scan-specific bright-material detector) gates every tier including A, so a mesh coherently snapped onto the *neighboring* wrap scores pap≈100/iqr≈0 and passes clean.
- **Effort:** hours each. **Impact:** unblocks literally everything else in this report — nothing downstream (P1.x, P2.x, or either judged finding) is measurable while these hold.

**4. [Plan cross-reference] P1.2 — wire `surf-consist` into the scorecard; it is the only computed signal of wrap identity, and it's currently thrown away.**
- Wrong-wrap capture is structurally invisible to every other metric (pap/iqr/ridge/delta don't encode *which* sheet). `surf-consist`'s wide-search escalation already *detects* coherent peaks at neighboring wraps but the outlier-rejection step at `surf_repair.hpp:214` discards the signal instead of surfacing it as a scorecard caution. `compose()` already accepts a `consist=` argument that nothing passes.
- **Integration sketch:** pass consist verdicts into `compose()`; OFFSET demotes the alignment tier; where no neighbor overlaps, mark alignment low-confidence rather than silently treating it as checked; surface the wide-search refusal as an explicit caution flag.
- Explicitly **skip** the alternative proposed elsewhere (winding-angle-monotonicity via umbilicus) — redundant with consist and needs extra machinery (umbilicus curves) this doesn't buy anything additional.
- **Effort:** ~1 day, depends on P0.1. **Impact:** high — this is the only near-term path to wrong-wrap detection, which the framing calls out as a problem with "no single-mesh oracle."

**5. [Plan cross-reference] P2.2 threshold recalibration against a labeled anchor set — but substitute/complement hand-labeling with the fault-injection synthetic corruptions (idea #2) for the corruption-type thresholds specifically.**
- All current cuts (pap 92/85/70, ridge tiers, health cv/holes) were hand-set on ~2 segments. The plan's own diagnostic — "A:0 with pap≥92 & iqr≤5" — is a live signal today that the A gate is measuring estimator artifacts, not quality.
- Synthetic corruptions with known-by-construction ground truth are strictly cheaper and non-circular for calibrating thresholds tied to corruption *types* (how large must a uniform-offset detector's gate be to survive test (b)); hand-grading 15–20 segments spanning the 5 scrolls is still needed for the parts synthetic corruption can't stand in for (real-world texture/scan variation, actual auto_grown damage patterns).
- **Effort:** ~1 day, depends on P1.1–P1.4. **Impact:** high but sequenced last among the top 5 — nothing to calibrate until P0/P1 land.

Neither judged finding is architecture-changing — both are explicitly sequencing/gating proposals layered on the existing pipeline (crop_qc → scorecard → repair_corpus → feed.hpp), not new components or a different pipeline shape. Within the broader plan cross-reference, the closest thing to an architectural change is:

**Net: this cycle's synthesis is entirely gating/measurement/consolidation, not new pipeline architecture.** That's the correct call given the input — the adversarial plan's own insight #5 (grade boundaries sit inside sampling noise) makes any bigger architectural bet unmeasurable right now.

### (4) Do-not-pursue list

1. **Any CPD/SDF/MRF/graph-cut repair-operator investment before P0+P1.3 land.** Explicit thesis of finding #1: the re-grade that would judge such an operator's improvement is itself dominated by sampling noise (SE≈0.17 vs 0.4 threshold). Building fancier repair math now produces an unmeasurable "did it help."
2. **Winding-angle-monotonicity check via umbilicus as a wrap-identity signal.** Adversarial plan P1.2 explicitly rejects this as redundant with `surf-consist` while requiring extra machinery (umbilicus curves) that doesn't buy anything consist doesn't already give.
3. **Building structure-tensor sheetness pre-emptively for 7.91µm scrolls.** P2.1 is explicit: measure pap saturation first (hours); only build sheetness if the measurement shows pap stops discriminating. Building it now is speculative effort against an unconfirmed premise, and even if built it still can't see wrap identity (that stays P1.2/consist's job regardless).
4. **Treating the noisy-label ML literature (2503.12218, teacher-guided early-learning) as prescribing a new mechanism.** Both are network-training-time noisy-label methods; mapped onto our grading-crop problem they add citations to an argument already fully derived from our own binomial-sampling arithmetic. Do not adapt their training-loop correction algorithms literally — the applicable insight (uncertainty-aware, don't trust a point estimate near a threshold) is already P1.3.
4. **Re-running fault-injection as a standing gate without also making it CI-aware (Wilson-CI, idea #1).** Finding #2's own review flags this: a standing gate built on flat 8-crop point estimates just adds a second flaky gate stacked on the first. Do not ship the "standing gate" half of finding #2 in isolation from the CI-awareness of finding #1 — they must land together or the gate is theater.
5. **Grades-as-weights (soft-weighting training loss by grade) before the grade itself is denoised.** Not explicitly rejected in the source docs, but follows directly from the same logic as items 1 and 4 above: weighting an unreliable signal encodes the noise into training rather than removing it. Resolve grading reliability (P0–P1.3) first; revisit weighting vs. accept/reject only after.

Nothing architecture-level is warranted this cycle; the entire ROI is in fixing measurement/gating before any new repair-operator or model-in-the-loop investment.

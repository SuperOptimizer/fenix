# GT-quality system — adversarial improvement plan (2026-07-09)

> Produced by a 56-agent adversarial review workflow (5 finder lenses × adversarial
> verification, 2 refuters/finding, 25 findings → 21 confirmed → synthesis).
> Run: wf_548d2f35-823. Companion to gt-autograde-improve.md — this SUPERSEDES its
> threshold/grading details where they conflict.


## 1. Critical insights

1. **pap(ct>40) is a bright-material detector promoted to primary grade denominator.** It is one-sided (verifies "on some bright stuff", never "on THIS wrap"), single-voxel (surf_qc.hpp:292, no 3x3x3 mean unlike delta mode, tallied before the 3-tap smoothing), hardcoded (no CLI knob — unreachable by the planned per-scroll calibration), and scan-specific (calibrated on one 2.4µm/78keV/masked u8 scan, now the denominator across 5 scrolls). scorecard.py tiers B/C/D gate on pap ALONE. A mesh coherently snapped to the neighboring wrap scores pap≈100 AND iqr≈0 → passes even the A gate.
2. **Wrap identity is structurally invisible to every single-mesh metric.** pap, iqr, ridge%, delta — none encode which sheet the mesh is on. Only surf-consist can, and it is NOT wired into scorecard.py (compose() accepts `consist=` but grade_corpus/scorecard never pass it; doc step 5 unimplemented). The one signal that DID expose wrong-wrap capture (wide-search finding coherent peaks at neighboring wraps, refused by outlier rejection at surf_repair.hpp:214) is computed and discarded instead of surfaced as a scorecard caution.
3. **The iqr gate fails OPEN on exactly the meshes it can't measure.** iqr is computed only over peak-firing points (no-peak points `continue` before `offs.push_back`, surf_qc.hpp:307/321) — the conditional sample of the discredited-in-dense-regions classifier. Worse: `f64 iqr = 0` initialization with the `offs.size() > 4` guard means a near-zero-peak mesh reports iqr=0 = "perfectly tight". n_offs is never emitted, so no consumer can guard.
4. **Three divergent grading functions coexist and collide on disk.** grade_corpus.py grade() (pure ridge — doesn't even parse on-papyrus), scorecard.py align_verdict() (pap-primary, no med/coh/n_valid gates), and repair_corpus.py:102's inline `a_iqr<=4 → "A"`. All write/read /tmp/gtqc/scorecards; repair_corpus globs `{scroll}__*.json` which matches both `{seg}.json` and `{seg}.card.json` and KeyErrors on `c["fxsurf"]` for card files. Same segment → different letter depending on which script ran last.
5. **Grade boundaries sit inside sampling noise, frozen by seed=0.** 8 crops × k=80 → pap/ridge carry ±5-10 binomial noise; frac_crops_good over 8 Bernoulli trials has SE≈0.17 vs the D/E pivot at 0.4; repair_corpus's `improved = a_ridge > b_ridge+5` is ~1σ. Crop placement rejects <30%-valid windows, so margins — where auto_grown damage concentrates — are never sampled for the ALIGNMENT grade (health axis does catch geometric damage whole-mesh; the bias hits alignment only). Grades don't flip run-to-run only because rng seed 0 freezes the coin-flip.

Meta-insight: the measured repair gain (83→93% pap) is largely trustworthy — search=8 can't jump a wrap at 2.4µm and wide-search escalation was correctly refused — but the GRADING layer above it is currently a single fragile metric with three inconsistent consumers and no uncertainty accounting.

## 2. Prioritized actions (deduplicated)

### P0 — hours each, unblock everything else
- **P0.1 Single source of truth for grading.** Move align_verdict/health_verdict/compose into an importable module in `tools/labelqc/scorecard.py`; delete grade_corpus.py grade() and repair_corpus.py's inline `a_iqr<=4→A`; both scripts call the shared function. Keep grade_corpus's coh<25→E and n_valid<100k→E gates when merging (scorecard lacks both; they catch scrambled-normal/fragment garbage). Fix the filename collision: one card schema/filename; repair_corpus reads only that. *Effort: hours. No dependencies. Every other fix is meaningless while three graders disagree.*
- **P0.2 Emit n_offs; kill the fail-open iqr.** surf_qc.hpp profile mode: print `n_offs=offs.size()`; emit iqr as absent/-1 when offs.size()<=4 (never 0). crop_qc.py parses n_offs, drops low-n crops from iqr_med, carries n_offs_med into the card. scorecard refuses to gate on iqr when n_offs<40. *Effort: hours. Prereq for P0.3 and P1.2.*
- **P0.3 Close the pap-alone tier hole.** scorecard.py: require iqr<=6 or coherent>=70 at EVERY pap tier (not just A), plus |med_off|<=2 in the pap branch (blocks uniform neighbor-wrap snap grading A). *Effort: hours. Depends on P0.2 for the n_offs floor.*

### P1 — day each, make the primary metric valid
- **P1.1 Parameterize + localize the pap threshold.** Add `papthr=` to surf-qc (default 0 = auto = per-crop/per-mesh Otsu valley on the sampled profile population — reuse `preprocess::otsu_threshold_u8` + feed.hpp's unimodality guard, fallback floor when unimodal). Tally pap on the 3-tap-smoothed prof[W] (or the 3x3x3 `at_vox`) instead of one raw voxel. crop_qc.py already has the local CT box → nearly free per-crop. *Effort: day. Makes pap transportable across the 5 scrolls before grading the remaining 86 LOD-2 segments.*
- **P1.2 Wire surf-consist into the scorecard (doc step 5).** Pass consist verdicts into compose(); OFFSET demotes the alignment tier (not just CROSS→E) — whole-wrap registration error presents as OFFSET vs overlapping neighbors. Where no neighbor overlaps, mark the card's alignment axis low-confidence rather than pretending it was checked. Also surface the repair loop's wide-search outlier-rejection refusal as a scorecard caution flag (it's the only computed signal of coherent wrong-wrap capture). This is the ONLY path to wrap identity; skip the proposed winding-angle-monotonicity check (needs umbilicus, redundant with consist). *Effort: day. Depends on P0.1.*
- **P1.3 Uncertainty-aware grading + boundary-stratified crops.** crop_qc.py: Wilson CIs on pap_med/ridge_med/frac_crops_good; scorecard assigns a tier only when the CI excludes the nearest threshold, else "borderline" → adaptively double crops (up to ~32; crops are WAN-local and cheap). Prioritize resolving C/D (repair vs trust-mask) and D/E (trust-mask vs quarantine) — B/C ambiguity is low-consequence (both route through repair + re-grade). Add a 10-30%-valid boundary stratum with its own score. Replace repair_corpus's `+5 ridge` improved-gate with a CI-separated gate. *Effort: day. Depends on P0.2.*
- **P1.4 Dense-region dispersion substitute.** Where n_offs is low, gate A/tier on IQR of air-edge alpha-crossing offsets (already implemented — surf_qc's air_edge, exercised by repair mode=alpha) or profile-centroid offsets, instead of peak-offset IQR. *Effort: half-day. Depends on P0.2.*

### P2 — conditional / measure-first
- **P2.1 Measure pap saturation at 7.91µm before building sheetness.** Run surf-qc profile on one known-good and one known-bad 7.91µm segment; if pap no longer discriminates (partial-volume fills inter-sheet gaps >thr), that triggers the already-planned step-3 structure-tensor sheetness gate. Do NOT build it pre-emptively; note sheetness still can't see wrap identity (that stays P1.2's job). *Effort: hours to measure; days if triggered.*
- **P2.2 Threshold recalibration against a labeled anchor set.** All cuts (pap 92/85/70, ridge tiers, health cv/holes) were hand-set on ~2 segments; after P0/P1 land, hand-grade ~15-20 segments spanning the 5 scrolls and fit the cuts. A:0 with pap>=92&iqr<=5 is diagnostic that the A gate is currently measuring estimator artifacts, not quality. *Effort: day. Depends on P1.1-P1.4.*
- **P2.3 pap_void counter (cheap two-sidedness).** Emit `pap_void` = on-papyrus AND (void_left||void_right) within ±W (void flags already computed per point); cap embedded% at tiers B+ to catch papyrus-contact-embedded meshes riding pap>=85 to B. *Effort: hours. Nice-to-have after P1.1.*

## 3. Cheapest end-to-end validating experiment

**Deliberate-fault injection round-trip on ~5 known-good segments (half a day, zero new code beyond P0):** take 5 segments the session facts already establish as good (post-repair 93% pap, AGREE in consist), and synthetically corrupt copies four ways matching the taxonomy: (a) shift the whole mesh by one wrap spacing (~±8-14 vox along normals) = wrong-wrap; (b) constant +3 vox offset = repairable-uniform; (c) smooth sinusoidal per-uv warp ±4 vox = repairable-warp; (d) scramble 20% of margin points = local damage. Run the full loop — crop_qc → scorecard → repair_corpus → re-grade → feed.hpp rasterize with trust grids — on originals + corruptions.

Pass criteria: originals grade A/B; (b),(c) grade C-D pre-repair, recover to A/B post-repair; (d) trust-masks the margin tiles; and critically **(a) must NOT grade above the original** — under the current code it will grade A (pap≈100, iqr≈0), which is the single measurement that proves/disproves the whole finding set and quantifies exactly how much P0.3 + P1.2 buy. This one experiment exercises every oracle, the repair operator, the grade composition, and the training consumer, with ground truth known by construction — no hand-labeling required.
